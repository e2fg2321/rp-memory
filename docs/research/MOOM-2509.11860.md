# MOOM: Maintenance, Organization and Optimization of Memory in Ultra-Long Role-Playing Dialogues

**arXiv:** 2509.11860 (September 2025)
**Authors:** Weishu Chen, Jinyi Tang, Zhouhui Hou, Shihao Han, Mingjie Zhan, et al.
**Affiliation:** Beijing University of Posts and Telecommunications / SenseTime

## Core Problem
How to extract, organize, and retain the right information over ultra-long RP conversations (600+ turns) without unbounded memory growth.

## Architecture: Dual-Branch Memory Plugin

### Branch 1: Narrative Summarization Branch (NSB)
- Hierarchical, multi-timescale summarization grounded in literary/narrative theory
- Dialogue turns packaged into first-level information units (threshold: 6 turns)
- Units iteratively summarized at higher levels (threshold: 5 per level)
- Creates a hierarchy: fine-grained recent events → coarse-grained older events
- Mirrors human memory: recent = detailed, old = broad strokes

### Branch 2: Persona Construction Branch (PCB)
- Builds and maintains structured character profiles
- Three merging strategies:
  - **Rule-based:** Simple replaceable/appendable attributes (e.g. location)
  - **Embedding-based:** BGE similarity scores for contradictory keys
  - **LLM-based:** Complex semantic reconciliation for conflicting character details

### Forgetting Mechanism (Competition-Inhibition)
- Inspired by cognitive science "competition-inhibition" theory
- Three memory categories:
  - **R_c (Relevant & effective):** Both relevant and useful
  - **N_c (Interfering/noisy):** Relevant but contradictory
  - **U_c (Unactivated):** Not accessed recently
- Scoring: S = alpha * sigmoid(gamma * (rc - b)) + beta * sum(1/(rc - r + epsilon))
- Parameters: alpha=0.1, beta=0.9, k=9
- Memories that are frequently retrieved survive; old unaccessed decay
- Modulated by retrieval history, NOT simple time-based Ebbinghaus decay

## Dataset: ZH-4O
- 28 dialogue sessions averaging 600 turns each
- 1,115 manual memory annotations
- 1,068 multiple-choice probe questions
- Chinese language

## Key Results
- MOOM-72B: BERTScore 0.8071, QA Precision 0.840 (vs Mem0-72B: 0.7108, 0.612)
- Most efficient method (fewest LLM calls)
- Competition-inhibition forgetting > Ebbinghaus curve (MemoryBank)
- At 6k token budget, MOOM > competitors at 10k budget
- Ablation: NSB + PCB combined >> either alone
- 9 retrieved items match/beat full-context approaches

## Key Takeaways for rp-memory
1. **Dual representation** (facts vs narrative) is synergistic — both needed
2. **Active forgetting** via retrieval-frequency weighting > simple time decay
3. **Hierarchical temporal compression** is practical and effective
4. **Memory efficiency** with good structure: 9 items can beat brute-force full context
5. **Embedding-based merging** for conflict resolution is a proven approach
